## 1. 什么是进程？什么是线程？二者有什么区别？

参考答案：

进程是操作系统进行资源分配和管理的基本单位，拥有独立的地址空间、代码、数据、堆、文件等资源。线程是进程中的一个顺序执行流，是 CPU 调度执行的基本单位。一个进程可以包含多个线程，同一进程内的线程共享进程的代码和堆等资源，但每个线程有自己的栈、程序计数器和运行状态。进程之间资源隔离更强，线程之间通信更方便但也更容易产生共享数据冲突。

易错点：不要只写“线程比进程小”，要写出资源共享和独立执行流。

## 2. Java 多线程程序有什么特点和优点？

参考答案：

Java 在语言和类库层面支持多线程，核心类是 `Thread`，也可以通过 `Runnable` 定义线程体。多线程程序允许多个任务在同一进程中并发执行，例如一个线程进行计算，另一个线程等待输入或处理 IO。优点是提高程序响应性和系统资源利用率，也能让程序结构更贴近真实的并发任务。Java 线程最终会映射到操作系统线程，具体调度由 JVM 和操作系统共同决定。

易错点：多线程不一定让单个任务更快，它主要提升并发处理能力和响应性。

## 3. Java 创建线程有哪些方式？`start()` 和 `run()` 有什么区别？

参考答案：

Java 创建线程常见两种方式：第一，继承 `Thread` 类并重写 `run()` 方法，然后创建子类对象调用 `start()`；第二，实现 `Runnable` 接口并实现 `run()` 方法，再把该对象传给 `new Thread(runnable)`，最后调用 `start()`。`start()` 的作用是启动新线程，使线程进入可运行状态，由调度器安排执行；`run()` 只是普通方法，直接调用不会产生新线程，而是在当前线程中顺序执行。

易错点：考试中如果问“为什么不用直接调用 run”，答案就是直接调用不会启动新线程。

## 4. 多线程会带来哪些问题？如何解决？

参考答案：

多线程会带来执行顺序不确定、共享数据竞争、数据不一致、死锁和可见性问题。当多个线程同时访问并修改同一份共享数据时，如果没有同步控制，结果可能取决于线程调度顺序。解决方法包括把共享数据设为 `private`，所有访问都通过同步方法或同步代码块进行；使用 `synchronized` 建立临界区；线程协作时使用 `wait()`、`notify()`、`notifyAll()`；访问多个共享资源时保持统一加锁顺序，避免死锁。

易错点：同步不是让程序“按固定顺序执行”，而是保护共享数据的一致性。

## 5. 什么是对象锁？`synchronized` 有什么作用？

参考答案：

Java 中每个对象都可以作为锁对象，与 `synchronized` 配合形成对象锁。线程进入某个对象的同步方法或同步代码块前，必须先获得该对象的锁；获得锁后，其他线程不能同时进入同一对象受该锁保护的临界区。`synchronized` 的作用是保证同一时刻只有一个线程执行临界区代码，从而避免多个线程同时修改共享数据导致不一致。

易错点：`public synchronized void m()` 等价于 `synchronized(this)` 包住整个方法体。

## 6. `wait()`、`notify()`、`notifyAll()` 的作用分别是什么？

参考答案：

`wait()` 使当前线程释放对象锁，并进入该对象的等待池，直到被其他线程唤醒或中断。`notify()` 唤醒等待池中的一个线程，使其进入锁池等待重新获得对象锁。`notifyAll()` 唤醒等待池中的所有线程。它们必须在 `synchronized` 方法或代码块中调用，因为它们依赖对象锁来完成线程协作。

易错点：`notify()` 唤醒线程后，并不代表被唤醒线程马上运行，它还要重新竞争对象锁。

## 7. `sleep()` 和 `wait()` 有什么区别？

参考答案：

`sleep()` 是 `Thread` 类的静态方法，用于让当前线程暂停一段时间，到时后重新进入可运行状态；它不会释放当前持有的对象锁。`wait()` 是 `Object` 类的方法，必须在同步代码中调用，调用后当前线程会释放对象锁并进入等待池，等待其他线程通过 `notify()` 或 `notifyAll()` 唤醒。简单说，`sleep()` 主要用于暂停执行，`wait()` 主要用于线程间协作。

易错点：是否释放锁是二者最重要区别。

## 8. 什么是死锁？如何避免死锁？

参考答案：

死锁是指两个或多个线程互相等待对方持有的资源，导致所有相关线程都无法继续执行。例如线程 A 持有锁 1 等待锁 2，线程 B 持有锁 2 等待锁 1。避免死锁的常见方法是减少锁嵌套，缩小同步范围；如果必须获取多个锁，应在整个程序中规定统一的加锁顺序，并按相反顺序释放锁；也可以设置超时或改用更高层的并发工具。

易错点：只写“使用 synchronized”不是避免死锁，错误使用多个锁反而可能造成死锁。

## 9. 什么是线程状态？Java 线程大致有哪些状态变化？

参考答案：

线程创建后处于新建状态，调用 `start()` 后进入可运行状态，调度器选中后进入运行状态。线程运行过程中可能因为 `sleep()`、`join()`、等待对象锁或调用 `wait()` 进入阻塞或等待状态。`run()` 方法执行完毕后线程进入终止状态，不能再次启动。线程状态变化由程序方法调用、锁竞争和调度器共同决定。

易错点：调用 `start()` 不代表马上运行，只表示可以参与调度。

## 10. Java GUI 中 AWT 的基本架构是什么？

参考答案：

AWT 位于 `java.awt` 包，是 Java 早期的 GUI 工具包。AWT 组件多为重量级组件，依赖本地平台的 peer 实现。它的基本结构包括组件、容器、布局管理器和事件处理模型。`Component` 是可显示组件的父类，`Container` 可以包含其他组件，`Frame`、`Dialog`、`Window` 是常见顶层窗口，`Panel` 常用于组织一组组件。

易错点：AWT 不只是几个控件名称，答题要写出组件、容器、布局和事件模型。

## 11. Swing 和 AWT 有什么关系？Swing 的层次结构有什么特点？

参考答案：

Swing 位于 `javax.swing` 包，建立在 AWT 的事件模型之上，是更高级的 GUI 工具包。Swing 大多数组件是轻量级组件，不直接依赖本地平台 peer，因此界面表现更统一。常见顶层容器是 `JFrame`，普通容器常用 `JPanel`，常见组件有 `JButton`、`JLabel`、`JTextField` 等。Swing 顶层容器内部通常有 Root Pane 结构，包括 `glassPane`、`layeredPane`、`contentPane` 和可选菜单栏。

易错点：Swing 不是完全替代 AWT，它仍然使用 AWT 的事件模型和部分基础类。

## 12. GUI 事件处理模型包括哪些角色？处理步骤是什么？

参考答案：

GUI 事件处理模型包括事件源、事件对象和事件监听器。事件源是发生事件的组件，例如按钮；事件对象保存事件信息，例如 `ActionEvent`；监听器是处理事件的对象，例如 `ActionListener`。基本步骤是创建组件，定义监听器，实现事件处理方法，把监听器注册到事件源上，例如调用 `button.addActionListener(listener)`，当事件发生时系统自动回调监听器方法。

易错点：不要把“点击按钮后执行代码”只写成一句话，要写清事件源、事件对象、监听器和注册。

## 13. Java GUI 中监听器有哪些定义方式？

参考答案：

监听器可以通过多种方式定义。第一，单独定义一个类实现监听器接口；第二，让当前窗口类实现监听器接口；第三，使用内部类或匿名内部类；第四，对只有一个抽象方法的监听器可以使用 lambda 表达式；第五，对于方法较多的监听器接口，可以继承适配器类，例如 `WindowAdapter`、`MouseAdapter`，只重写需要的方法。

易错点：`ActionListener` 可用 lambda，因为它是函数式接口；`MouseListener` 方法多，常用适配器简化。

## 14. 什么是 Layout？常见布局管理器有哪些？

参考答案：

Layout 是布局管理器，用于决定容器中组件的位置和大小。Java GUI 不推荐手动设置绝对坐标，而是通过布局管理器适配不同窗口大小和平台。常见布局包括 `FlowLayout`，按顺序从左到右排列；`BorderLayout`，分为东、西、南、北、中五个区域；`GridLayout`，按规则网格排列；`CardLayout`，用于多页面切换；`GridBagLayout`，功能强但复杂，适合复杂表单。

易错点：复杂界面通常不是只用一个布局，而是多个 `Panel/JPanel` 嵌套不同布局。

## 15. Infinity 和 NaN 在 IEEE 754 中如何定义？

参考答案：

在 IEEE 754 浮点数表示中，Infinity 和 NaN 都是特殊值。正负无穷的指数位全为 1，尾数部分为 0，符号位决定是正无穷还是负无穷。NaN 表示 Not a Number，指数位全为 1，尾数部分非 0。Java 的 `float` 和 `double` 遵循 IEEE 754，提供 `Float.NaN`、`Double.NaN`、`Double.POSITIVE_INFINITY`、`Double.NEGATIVE_INFINITY` 等常量。

易错点：NaN 不是普通数字，也不是无穷大。

## 16. NaN 的 Golden Rule 是什么？Java 中如何判断 NaN？

参考答案：

NaN 的 Golden Rule 可以概括为：NaN 是无序的，任何包含 NaN 的普通大小比较都为 `false`，包括 `==`，唯一特殊的是 `!=` 为 `true`。因此 `Double.NaN == Double.NaN` 的结果是 `false`。在 Java 中判断一个值是否为 NaN，不应该使用 `==`，而应该使用 `Double.isNaN(x)` 或 `Float.isNaN(x)`。

易错点：`x == Double.NaN` 永远不是判断 NaN 的正确写法。

## 17. 浮点数除以 0 和整数除以 0 有什么区别？

参考答案：

在 Java 中，整数除以 0 会抛出 `ArithmeticException`，因为整数没有无穷大或 NaN 的表示。而浮点数遵循 IEEE 754，浮点数除以 0 不一定抛异常，例如 `1.0 / 0.0` 得到 `Infinity`，`1.0 / -0.0` 得到 `-Infinity`，`0.0 / 0.0` 得到 `NaN`。另外，`0.0 == -0.0` 的结果是 `true`，但它们参与除法时可能产生不同符号的无穷大。

易错点：要区分 `1 / 0` 和 `1.0 / 0.0`。

## 18. Java 枚举类型如何定义和使用？

参考答案：

枚举用 `enum` 定义，用于表示一组固定的具名值。例如 `enum Week { MONDAY, TUESDAY, WEDNESDAY }`。枚举常量本质上是该枚举类型的 `static final` 实例，枚举类型隐式继承 `java.lang.Enum`，因此不能再继承其他类，但可以实现接口。枚举常用方法有 `values()`、`name()` 和 `ordinal()`，也可以在 `switch` 语句中使用枚举常量。

易错点：枚举不是 `int` 常量集合，而是一种特殊的类。

## 19. 枚举中能否定义字段、构造方法和成员方法？

参考答案：

可以。枚举类型可以定义字段、构造方法和成员方法，用于给每个枚举常量附加属性。例如硬币枚举可以定义面值字段 `value`，并通过构造方法初始化。枚举构造方法只能是 `private` 或省略访问修饰符，不能由外部通过 `new` 调用，因为枚举常量在枚举类加载时已经固定创建。若枚举常量后面还有字段或方法，常量列表后要写分号。

易错点：枚举构造方法不能写成 `public`。

## 20. Java 程序中类的静态初始化顺序是什么？

参考答案：

类第一次主动使用时会触发类加载和初始化。静态字段和静态初始化块按照它们在类中出现的先后顺序执行，且每个类只执行一次静态初始化。如果初始化子类，通常要先初始化父类，再初始化子类。静态字段在显式赋值前会先有默认值，因此在复杂继承初始化中，如果父类静态块过早访问子类静态字段，可能读到默认值。

易错点：静态初始化不是每次创建对象都执行，而是类加载时执行一次。

## 21. Java 创建对象时实例初始化顺序是什么？

参考答案：

创建对象时，首先为对象分配内存，并把实例字段设为默认值。然后调用父类构造链，从 `Object` 到直接父类依次完成父类初始化。接着执行本类实例字段显式初始化和实例初始化块，最后执行本类构造方法体，并返回对象引用。如果是子类对象，父类部分一定先于子类部分完成构造。

易错点：字段显式初始化不是最先发生，最先是默认值初始化。

## 22. 为什么不建议在构造方法中调用可被子类重写的方法？

参考答案：

因为父类构造方法执行时，如果调用一个可被重写的方法，会发生动态绑定，实际调用子类重写后的方法。但此时子类字段还没有完成显式初始化，可能仍是默认值，从而产生意料之外的结果。项目中的 `PolyConstruct.java` 就体现了这个问题：父类构造中调用 `draw()`，实际执行子类 `draw()`，但子类字段还没初始化，所以输出默认值。

易错点：构造期间依然会发生动态绑定，这是很多读程题的陷阱。

## 23. 方法重载和方法重写有什么区别？

参考答案：

方法重载发生在同一个类或继承关系中，要求方法名相同但参数列表不同，返回值不能作为区分重载的唯一依据，重载在编译期根据实参类型确定。方法重写发生在父子类之间，子类重新定义父类的实例方法，要求方法名、参数列表和返回类型规则匹配，运行时根据实际对象类型动态绑定。简单说，重载看参数，重写看对象。

易错点：`static` 方法没有真正的运行时多态，不能按普通实例方法重写来理解。

## 24. `static` 方法是否具有多态性？为什么？

参考答案：

`static` 方法不具有运行时多态性。静态方法属于类，不属于具体对象，调用时按引用变量的编译时类型或类名来确定版本。即使父类引用指向子类对象，调用静态方法时也会根据父类引用类型绑定到父类静态方法。普通实例方法才会根据实际对象类型进行动态绑定。

易错点：子类可以声明与父类同名的静态方法，但这叫隐藏，不是普通意义上的重写。

## 25. `private` 方法能否被重写？字段是否具有多态性？

参考答案：

`private` 方法不能被子类重写，因为它对子类不可见。子类中定义同名方法只是一个新的方法，不是对父类私有方法的重写。字段也不具有多态性，父类字段和子类同名字段是两个不同字段；字段访问看引用类型，方法调用才可能动态绑定。因此读程题中要区分字段、私有方法、静态方法和普通实例方法。

易错点：不是所有“同名”都叫重写。

## 26. `final` 关键字可以修饰什么？含义分别是什么？

参考答案：

`final` 可以修饰变量、方法和类。修饰变量表示变量只能赋值一次，基本类型的值不能变，引用类型的引用不能再指向其他对象，但对象内部状态可能还能改变。修饰方法表示该方法不能被子类重写。修饰类表示该类不能被继承。空白 final 字段必须在声明处或所有构造方法中完成初始化。

易错点：`final` 引用不可变，不等于引用指向的对象内容一定不可变。

## 27. Java 异常体系中 Error、Exception、RuntimeException 有什么区别？

参考答案：

`Throwable` 是 Java 异常体系的根类，主要分为 `Error` 和 `Exception`。`Error` 表示严重错误，通常与 JVM 或系统资源有关，应用程序一般不处理。`Exception` 表示程序可以处理的异常。`RuntimeException` 及其子类属于 unchecked exception，编译器不强制捕获或声明；除 `RuntimeException` 外的多数 `Exception` 属于 checked exception，编译器要求用 `try-catch` 捕获或用 `throws` 声明抛出。

易错点：不是所有异常都必须捕获，checked exception 才被编译器强制处理。

## 28. `throw` 和 `throws` 有什么区别？

参考答案：

`throw` 是语句，用于在方法体中主动抛出一个具体的异常对象，例如 `throw new IOException("错误")`。`throws` 是方法声明的一部分，用于说明该方法可能把某些异常交给调用者处理，例如 `public void read() throws IOException`。一个是实际抛出，一个是声明可能抛出。checked exception 如果在本方法中不捕获，就必须通过 `throws` 声明。

易错点：`throw` 后面跟异常对象，`throws` 后面跟异常类型列表。

## 29. `try-catch-finally` 的执行规则是什么？

参考答案：

`try` 块中放可能产生异常的代码。发生异常后，程序会查找匹配的 `catch` 块并执行；如果没有异常，则跳过 `catch`。`finally` 块无论是否发生异常通常都会执行，常用于关闭文件、网络连接等资源。如果 `try` 中异常被捕获，执行完 `catch` 和 `finally` 后程序继续向下执行；如果异常未被捕获，执行 `finally` 后异常继续向外传播。

易错点：多个 `catch` 要把子类异常写在父类异常前面，否则更具体的异常分支可能不可达。

## 30. Java IO 中字节流、字符流、缓冲流和数据流分别适合什么场景？

参考答案：

字节流以字节为单位读写，父类是 `InputStream` 和 `OutputStream`，适合复制图片、音频、任意二进制文件，也可处理文本但不方便编码。字符流以字符为单位读写，父类是 `Reader` 和 `Writer`，适合处理文本文件。缓冲流包装在节点流外部，通过减少实际 IO 次数提高效率，例如 `BufferedReader`、`BufferedWriter`，其中 `BufferedReader` 提供 `readLine()`。数据流 `DataInputStream` 和 `DataOutputStream` 适合按基本数据类型读写，读写顺序必须一致。

易错点：`File` 类只表示文件或目录路径及元信息，不负责读取或写入文件内容。
