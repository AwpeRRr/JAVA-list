# 开心农场程序第四次修改说明

## 一、任务要求

老师好，我这次是在原来的 `HappyFarmTest_ArrayList` 项目基础上进行修改。原程序已经能够使用二维 `ArrayList` 管理农场中的对象，包括小麦、玉米、鸡和牛，但是程序退出以后，已经创建的农场内容会丢失。

本次修改需要解决三个问题：

1. 将当前农场状态保存到本地文件 `farm.txt` 中。
2. 程序下一次启动时，能够读取 `farm.txt`，恢复上次保存的二维农场状态。
3. 测试类继续负责与用户交互，让用户能够创建、查看、照料、删除和保存对象。

## 二、原有程序结构

原程序中的类及功能如下：

| 类名 | 功能 |
| --- | --- |
| `FarmObject` | 所有农场对象的父类，保存编号、名称、类型等共同属性 |
| `Wheat` | 小麦对象，实现小麦的照料行为 |
| `Corn` | 玉米对象，实现玉米的照料行为 |
| `Chicken` | 鸡对象，实现鸡的照料行为 |
| `Cow` | 牛对象，实现牛的照料行为 |
| `Farm` | 使用 `ArrayList<ArrayList<FarmObject>>` 保存整个农场 |
| `FarmTest` | 主程序和菜单交互类 |

二维 `ArrayList` 中，外层集合表示农场的行，内层集合表示这一行中保存的对象。例如：

```text
第 0 行：小麦、鸡
第 1 行：牛
```

在内存中相当于：

```java
ArrayList<ArrayList<FarmObject>> farmRows;
```

## 三、我的修改思路

我没有改变原程序的对象继承结构，而是在原有设计上增加文件持久化功能。

我的处理流程是：

1. 用户在程序中创建和管理对象，数据仍然保存在二维 `ArrayList` 中。
2. 用户选择保存时，将农场行数和每一个对象的信息写入 `farm.txt`。
3. 程序下一次启动时，先判断是否存在 `farm.txt`。
4. 如果存在存档文件，就读取文件内容，按照对象类型重新创建 `Wheat`、`Corn`、`Chicken` 或 `Cow` 对象，再放回原来的行中。
5. 如果不存在存档，或者存档内容错误，就提示用户创建一个新的农场。

这样既保留了原来面向对象和二维集合的结构，也实现了程序退出后数据不丢失。

## 四、`farm.txt` 文件格式设计

我设计的保存文件是文本文件，编码为 UTF-8，内容示例如下：

```text
ROWS	2
OBJECT	0	Wheat	101	一号小麦
OBJECT	1	Cow	202	奶牛小花
```

每一部分的含义如下：

| 内容 | 含义 |
| --- | --- |
| `ROWS	2` | 当前农场共有 2 行 |
| `OBJECT` | 表示这一行记录的是一个农场对象 |
| `0` 或 `1` | 该对象位于二维集合的第几行 |
| `Wheat`、`Cow` | 对象真实的类名，用于读取时恢复具体对象 |
| `101`、`202` | 对象编号 |
| `一号小麦`、`奶牛小花` | 对象名称 |

选择保存类名，而不是只保存中文类型名称，是因为读取时可以明确判断应该执行：

```java
new Wheat(id, name);
new Cow(id, name);
```

这样恢复出来的对象仍然具有各自不同的 `care()` 行为。

## 五、`Farm.java` 中的改动

### 1. 增加文件操作所需的类

我在 `Farm.java` 中增加了文件读写相关的导入：

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
```

使用 `BufferedWriter` 和 `BufferedReader` 可以按行写入和读取文件；使用 `StandardCharsets.UTF_8` 是为了保证中文名称保存后不会乱码。

### 2. 增加保存方法 `saveToFile`

保存方法的定义如下：

```java
public void saveToFile(String fileName) throws IOException
```

它主要完成两步操作：

1. 先写入农场当前的行数。
2. 遍历二维 `ArrayList`，把每一个对象的行号、类名、编号和名称写入文件。

核心代码如下：

```java
writer.write("ROWS\t" + getRowCount());
writer.newLine();

for (int rowIndex = 0; rowIndex < farmRows.size(); rowIndex++) {
    ArrayList<FarmObject> row = farmRows.get(rowIndex);
    for (FarmObject obj : row) {
        writer.write("OBJECT\t" + rowIndex
                + "\t" + obj.getClass().getSimpleName()
                + "\t" + obj.getId()
                + "\t" + escapeText(obj.getName()));
        writer.newLine();
    }
}
```

这里的 `obj.getClass().getSimpleName()` 能够保存对象真正的类型，例如 `Cow` 或 `Wheat`。

### 3. 增加读取方法 `loadFromFile`

读取方法的定义如下：

```java
public boolean loadFromFile(String fileName) throws IOException
```

这个方法首先判断存档文件是否存在：

```java
if (!Files.exists(path)) {
    return false;
}
```

如果文件存在，程序先读取第一行中的农场行数，并建立新的二维集合：

```java
for (int i = 0; i < rowCount; i++) {
    loadedRows.add(new ArrayList<FarmObject>());
}
```

之后逐行读取对象记录。程序会取得对象所在行、类名、编号和名称，再创建相应对象：

```java
FarmObject obj = createObjectByClassName(fields[2], id, name, lineNumber);
loadedRows.get(rowIndex).add(obj);
```

### 4. 根据类型重新创建对象

为了满足“读取时根据文件中的对象类型重新创建对应对象”的要求，我增加了以下方法：

```java
private FarmObject createObjectByClassName(String className, int id, String name, int lineNumber)
        throws IOException {
    switch (className) {
        case "Wheat":
            return new Wheat(id, name);
        case "Corn":
            return new Corn(id, name);
        case "Chicken":
            return new Chicken(id, name);
        case "Cow":
            return new Cow(id, name);
        default:
            throw new IOException("第 " + lineNumber + " 行包含未知对象类型: " + className);
    }
}
```

例如存档中记录的是 `Cow`，读取后创建的一定是 `Cow` 对象，所以调用 `care()` 时，程序仍会输出给牛喂牧草和挤牛奶的内容，而不会丢失对象的具体行为。

### 5. 对文件内容进行校验

为了防止错误文件导致程序直接出现异常，我增加了以下校验：

- 农场行数必须大于 `0`。
- 对象所在行必须在农场行数范围以内。
- 对象编号必须为正数，并且不能重复。
- 对象名称不能为空。
- 对象类型必须是程序支持的四种类型之一。

我先把读取到的数据放到临时集合 `loadedRows` 中，只有所有数据读取正确以后，才替换当前农场内容：

```java
farmRows.clear();
farmRows.addAll(loadedRows);
```

这样如果存档中途出现错误，原有内存数据不会被读取到一半的内容覆盖。

### 6. 对对象名称中的特殊字符进行处理

因为保存文件使用制表符分隔字段，如果用户在名称中输入制表符或换行符，就可能破坏文件格式。因此我增加了：

```java
private String escapeText(String text)
private String unescapeText(String text, int lineNumber)
```

保存时会将特殊字符进行转义，读取时再恢复原内容，从而使存档格式更加稳定。

## 六、`FarmTest.java` 中的改动

### 1. 指定保存文件名

我在测试类中增加了常量：

```java
private static final String SAVE_FILE = "farm.txt";
```

这样程序中的保存和读取统一使用同一个文件名，后期修改文件名也更加方便。

### 2. 启动时自动读取存档

原程序每次启动都会直接要求用户输入农场行数。修改以后，主方法先尝试读取存档：

```java
if (!loadSavedFarm()) {
    initFarm(scanner);
}
```

运行效果是：

- 如果存在正确的 `farm.txt`，程序直接恢复原来的农场。
- 如果不存在 `farm.txt`，程序再让用户初始化新的农场。
- 如果文件内容损坏，程序会提示读取失败，并创建新的农场。

### 3. 菜单增加保存功能

我在菜单中增加了选项：

```text
8. 保存当前农场状态到 farm.txt
```

用户输入 `8` 后，会调用：

```java
farm.saveToFile(SAVE_FILE);
```

并提示保存成功或保存失败的原因。

### 4. 退出程序时询问是否保存

用户输入 `0` 退出时，程序会先询问：

```text
退出前是否保存当前农场状态到 farm.txt？(y/n):
```

如果选择 `y`，就会在退出前保存当前状态。这可以避免用户修改了农场以后忘记手动保存。

## 七、运行过程示例

### 第一次运行：创建并保存

例如我建立两行农场，并添加两个对象：

```text
第 0 行：编号 101，小麦，名称 Wheat-A
第 1 行：编号 202，牛，名称 Cow-B
```

选择菜单 `8` 保存后，生成的 `farm.txt` 为：

```text
ROWS	2
OBJECT	0	Wheat	101	Wheat-A
OBJECT	1	Cow	202	Cow-B
```

### 第二次运行：自动恢复

程序重新启动后，不需要再次输入农场行数，而是显示：

```text
已从 farm.txt 恢复上一次保存的农场状态。
```

并能输出：

```text
第 0 行，共 1 个对象
  位置 0 -> 编号: 101  类型: 农作物-小麦  名称: Wheat-A
第 1 行，共 1 个对象
  位置 0 -> 编号: 202  类型: 动物-牛      名称: Cow-B
```

我再对第 `1` 行第 `0` 个位置的对象执行照料操作，程序输出：

```text
正在给牛 [Cow-B] 喂牧草和挤牛奶...
```

这证明读取文件后恢复出来的对象确实是 `Cow` 类对象，而不只是保存了名称和编号。

## 八、我对本次修改的总结

通过这次修改，我进一步理解了以下知识点：

1. 二维 `ArrayList` 不仅可以在程序运行期间保存对象，也可以配合文件读写完成状态持久化。
2. 保存对象时，除了保存属性，还需要保存对象的实际类型，才能在读取时重新创建正确的子类对象。
3. 文件读取不能只考虑正常情况，还应该处理文件不存在、数据格式错误、编号重复等异常情况。
4. 测试类除了实现菜单功能，还应该对用户给出清晰提示，使程序操作过程更完整。

本次修改在保持原程序结构的基础上，实现了农场对象的保存和恢复，使程序能够在多次运行之间保留用户创建的数据。
