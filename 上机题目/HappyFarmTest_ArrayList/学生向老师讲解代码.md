# HappyFarmTest_ArrayList 项目讲解稿

## 一、开场介绍

老师好，我这次讲解的是 `HappyFarmTest_ArrayList` 项目。它是一个 Java 控制台版的“开心农场管理系统”，主要功能是让用户在命令行里创建农场行、添加农作物或动物、查看所有对象、按名称查找、照料对象、删除对象以及清空农场。

这个项目的重点不是界面，而是练习 Java 面向对象和 `ArrayList` 的使用。项目里用到了抽象类、继承、方法重写、多态、集合嵌套、输入校验和菜单循环。

## 三、整体设计思路

我把农场里的所有对象都抽象成同一种父类 `FarmObject`。不管是小麦、玉米、鸡还是牛，它们都有三个共同属性：

- `id`：对象编号
- `name`：对象名称
- `type`：对象类型

同时它们都有一个共同动作，就是“照料”，所以父类里定义了一个抽象方法：

```java
public abstract void care();
```

但是不同对象的照料方式是不一样的。比如小麦是浇水和除草，牛是喂牧草和挤牛奶，所以具体的照料内容交给子类重写。这样就体现了面向对象里的继承和多态。

农场的数据结构放在 `Farm.java` 里：

```java
private ArrayList<ArrayList<FarmObject>> farmRows = new ArrayList<ArrayList<FarmObject>>();
```

这里使用的是嵌套 `ArrayList`。外层 `ArrayList` 表示农场有多少行，内层 `ArrayList` 表示某一行里放了哪些农场对象。比如 `farmRows.get(0).get(1)` 就表示第 0 行第 1 个位置的对象。

## 四、`FarmObject` 抽象父类讲解

`FarmObject` 是所有农场对象的父类。它的主要作用有三个：

第一，统一保存公共属性。小麦、玉米、鸡、牛都需要编号、名称和类型，所以这三个字段放在父类里，避免每个子类重复写。

第二，提供公共方法。比如 `getId()`、`getName()`、`getType()` 都是读取属性的方法，所有子类都可以直接继承使用。

第三，规定统一行为。`care()` 被定义成抽象方法，表示所有农场对象都必须有“照料”这个动作，但是具体怎么照料由子类自己决定。

`toString()` 方法也在父类里重写了，所以输出任何农场对象时，都可以统一显示编号、类型和名称。

## 五、四个子类讲解

`Wheat`、`Corn`、`Chicken`、`Cow` 四个类都继承自 `FarmObject`。

以 `Cow.java` 为例：

```java
public class Cow extends FarmObject {
    public Cow(int id, String name) {
        super(id, name, "动物-牛");
    }

    @Override
    public void care() {
        System.out.println("正在给牛 [" + getName() + "] 喂牧草和挤牛奶...");
    }
}
```

构造方法里使用 `super(id, name, "动物-牛")` 调用父类构造方法，把编号、名称和类型传给父类保存。

`care()` 方法使用 `@Override` 表示重写父类的抽象方法。这样当程序拿到一个 `FarmObject` 类型的变量时，即使它实际指向的是 `Cow` 对象，调用 `care()` 时也会执行牛自己的照料逻辑。

其他三个类也是同样的结构：

- `Wheat`：类型是“农作物-小麦”，照料方式是浇水和除草
- `Corn`：类型是“农作物-玉米”，照料方式是施肥和松土
- `Chicken`：类型是“动物-鸡”，照料方式是喂饲料和打扫鸡舍
- `Cow`：类型是“动物-牛”，照料方式是喂牧草和挤牛奶

## 六、`Farm` 农场管理类讲解

`Farm` 是项目的核心管理类，它不负责读取键盘输入，只负责管理数据。

### 1. 初始化农场

```java
public void initFarm(int rowCount) {
    farmRows.clear();
    for (int i = 0; i < rowCount; i++) {
        farmRows.add(new ArrayList<FarmObject>());
    }
}
```

这个方法会先清空原来的农场数据，然后根据用户输入的行数创建对应数量的空行。每一行本质上都是一个 `ArrayList<FarmObject>`。

### 2. 判断行号和位置是否合法

```java
public boolean isValidRowIndex(int rowIndex) {
    return rowIndex >= 0 && rowIndex < farmRows.size();
}
```

行号必须从 0 开始，并且小于农场总行数。

```java
public boolean isValidPosition(int rowIndex, int positionIndex) {
    return isValidRowIndex(rowIndex)
            && positionIndex >= 0
            && positionIndex < farmRows.get(rowIndex).size();
}
```

位置判断更严格，必须先保证行号存在，再判断这一行里的位置编号是否存在。这样可以避免数组越界或集合越界异常。

### 3. 添加对象

```java
public boolean addObject(FarmObject obj, int rowIndex) {
    if (obj == null || !isValidRowIndex(rowIndex)) {
        return false;
    }
    farmRows.get(rowIndex).add(obj);
    return true;
}
```

添加对象时，程序先判断对象是否为空、行号是否合法。如果都没问题，就把对象添加到指定行的末尾。

### 4. 删除对象

```java
public FarmObject removeObject(int rowIndex, int positionIndex) {
    if (!isValidPosition(rowIndex, positionIndex)) {
        return null;
    }
    return farmRows.get(rowIndex).remove(positionIndex);
}
```

删除对象时，如果位置不存在就返回 `null`；如果存在，就删除并返回被删除的对象。这样主程序可以根据返回值判断删除是否成功。

### 5. 查找对象

```java
public void searchByName(String name) {
    boolean found = false;

    for (int rowIndex = 0; rowIndex < farmRows.size(); rowIndex++) {
        ArrayList<FarmObject> row = farmRows.get(rowIndex);
        for (int positionIndex = 0; positionIndex < row.size(); positionIndex++) {
            FarmObject obj = row.get(positionIndex);
            if (obj.getName().equals(name)) {
                ...
            }
        }
    }
}
```

查找时使用两层循环。外层遍历每一行，内层遍历这一行里的每一个对象。如果名称相同，就输出对象的信息和它所在的位置。

这个查找是线性查找，优点是简单，适合当前练习项目；如果以后数据量很大，可以考虑用 `HashMap` 按编号或名称建立索引。

## 七、`FarmTest` 主程序讲解

`FarmTest.java` 是程序入口，里面有 `main()` 方法。

程序一开始创建一个 `Scanner` 读取键盘输入，然后调用 `initFarm(scanner)` 让用户输入农场行数。初始化完成后，进入 `while (running)` 循环，不断显示菜单。

菜单选项通过 `switch` 判断：

- 输入 `1`：重新初始化农场行数
- 输入 `2`：创建并添加农场对象
- 输入 `3`：输出所有对象和总数
- 输入 `4`：按名称查找对象
- 输入 `5`：照料指定位置的对象
- 输入 `6`：删除指定位置的对象
- 输入 `7`：清空农场对象
- 输入 `0`：退出系统

这样写的好处是结构清楚，每个菜单功能都被拆成了单独的方法，`main()` 不会特别乱。

## 八、创建对象的流程

创建对象的主要方法是 `createFarmObject(scanner)`。

我按这个顺序处理用户输入：

1. 让用户选择对象类型：小麦、玉米、鸡或牛。
2. 输入对象编号，并用 `farm.hasId(id)` 检查编号是否重复。
3. 输入对象名称，并判断名称不能为空。
4. 输入要添加到哪一行，并判断行号是否合法。
5. 调用 `createObjectByType(typeChoice, id, name)` 创建具体对象。
6. 调用 `farm.addObject(newObj, rowIndex)` 添加到农场。
7. 输出添加成功和对象所在位置。

`createObjectByType()` 可以看成一个简单的对象工厂：

```java
private static FarmObject createObjectByType(String typeChoice, int id, String name) {
    switch (typeChoice) {
        case "1":
            return new Wheat(id, name);
        case "2":
            return new Corn(id, name);
        case "3":
            return new Chicken(id, name);
        case "4":
            return new Cow(id, name);
        default:
            return null;
    }
}
```

虽然返回类型是 `FarmObject`，但是实际创建出来的可能是 `Wheat`、`Corn`、`Chicken` 或 `Cow`。这就是多态的体现。

## 九、照料对象时的多态体现

照料对象的方法是 `careForObject(scanner)`。

程序先根据用户输入的行号和位置编号，从农场里取出对象：

```java
FarmObject obj = farm.getObject(rowIndex, positionIndex);
```

这里变量类型是 `FarmObject`，但是它实际指向哪个子类对象，要看用户之前添加的是什么。

然后调用：

```java
obj.care();
```

如果这个对象是小麦，就执行小麦的 `care()`；如果是牛，就执行牛的 `care()`。主程序不需要写很多 `if` 判断对象类型，这就是使用继承和多态的好处。

## 十、输入校验设计

项目里有两个方法专门处理数字输入：

```java
private static int readInt(Scanner scanner, String prompt)
```

这个方法不断读取用户输入，并用 `Integer.parseInt()` 转成整数。如果输入的不是整数，就捕获 `NumberFormatException`，提示用户重新输入。

```java
private static int readPositiveInt(Scanner scanner, String prompt)
```

这个方法是在 `readInt()` 的基础上继续判断数字必须大于 0，主要用于对象编号这种必须为正数的输入。

这样做可以让其他业务方法更简洁，不用每个地方都重复写异常处理。

## 十一、可以向老师强调的知识点

这份代码主要体现了下面几个 Java 知识点：

- 使用 `ArrayList` 动态保存对象，数量不固定时比普通数组更灵活。
- 使用 `ArrayList<ArrayList<FarmObject>>` 表示二维结构。
- 使用抽象类提取公共属性和公共行为。
- 使用继承让小麦、玉米、鸡、牛复用父类代码。
- 使用方法重写让不同对象有不同的照料方式。
- 使用多态让 `FarmObject obj` 可以指向不同子类对象。
- 使用 `Scanner` 实现控制台交互。
- 使用 `switch` 实现菜单选择。
- 使用输入校验避免无效数字、空名称、重复编号和非法位置。

## 十二、项目目前的不足和改进方向

如果老师问我这个项目还能怎么改进，我会这样回答：

1. 目前数据只保存在内存里，程序退出后数据就没了，以后可以加入文件保存和读取功能。
2. 查找名称时是遍历所有对象，数据多的时候效率不高，可以考虑用 `HashMap` 优化。
3. 删除对象后，同一行后面的对象位置编号会自动前移，这一点可以在用户提示里说明得更清楚。
4. 目前只能按名称查找，以后可以增加按编号查找、按类型统计等功能。
5. `.project` 文件里有 Git 合并冲突标记，正式提交前需要清理。
6. 如果要让项目更规范，可以把源码放到 `src` 目录，并把编译后的 `.class` 文件放到 `bin` 或 `out` 目录。

## 十三、课堂汇报版总结

老师，整体来说，我这个项目是一个使用 `ArrayList` 实现的简单农场管理系统。`FarmTest` 负责用户交互，`Farm` 负责保存和管理数据，`FarmObject` 抽象出所有农场对象的共同属性和行为，四个子类分别表示小麦、玉米、鸡和牛。

我认为这个项目最核心的地方有两个：第一个是用嵌套 `ArrayList` 表示农场的多行结构，第二个是用抽象类和多态实现不同农场对象的照料行为。通过这个项目，我练习了集合、类的继承、方法重写、控制台菜单和输入校验这些知识点。
